Einführung Was enthält eine Vorlage und wofür kann man dies nutzen

Vielen Dank, dass Sie OpenCart Vorlage gekauft haben. Diese Dokumentation besteht aus einigen Bereichen und umfasst den ganzen Installationsprozess und Einstellungen der OpenCart Website bei Null. Wir haben unser Bestes gegeben, um diese Anleitung verständlich und einfach in der Handhabung zu machen.

Was ist denn OpenCart?

OpenCart ist eine kostenlose Plattform für Internethandel mit Open-Source für Internet-Unternehmer. OpenCart ermöglicht professionelle und sichere Basis für einen erfolgreichen Online-Shop zu erstellen. Gemeinschaft Opencart hat eine ganze Menge der Benutzer. Es sind sowohl erfahrene Webentwickler, die ein bequemes Interface suchen, als auch Shop-Besitzer, die zum ersten Mal Online-Business starten. OpenCart bietet verschiedene Funktionen, die ermöglichen, völlig den Online-Shop zu verwalten. Mit OpenCart Werkzeugen können Sie einfach Potentials Ihres Online-Shops entfalten. Details

Dateistruktur

Die Vorlage besteht aus folgenden Komponenten:

  • documentation - enthält ausführliche Anleitung zur Installation der Vorlage und Engine;
    • documentation.html - Hauptdatei der Dokumentation. Sie lesen sie jetzt :)
  • screenshots - enthält Bildschirmfotos des Themas;
  • sources - Quelldatei der Vorlage für Adobe Photoshop(.psd);
  • themeXXX - enthält:
    • themeXXX(full).zip - Archiv mit der OpenCart Engine, Vorlage und Demodaten des Themas (es ist der Installation bei Null gewidmet);
    • themeXXX(update).zip - Vorlagenarchiv ohne Demodaten (es ist der Installation auf den existierenden Online-Shop gewidmet);
    • themeXXX(extensions).zip - Datei mit XML-Dateien;
    • unzip.php - Datei, die zum Entpacken .zip nach dem Hochladen auf dem Hosting-Server benutzt wird;
  • info.txt - stellt die Information über Archive zur Verfügung;
  • fonts_info.txt - enthält die Information über Schriften, die in der Vorlage benutzt werden;

Vorbereitung Was soll man vor dem Arbeitsbeginn machen

Bevor Sie anfangen, OpenCart Website zu installieren, müssen Sie alles fertig machen.

Software

Um mit der Opencart Vorlage vollwertig zu arbeiten, empfehlen wir Ihne alle notwendigen Anwendungen herunterzuladen. Die Liste der notwendigen Software gibt es auf der Vorschauseite der Vorlage. Anforderungen für verchiedene Vorlagen können sich ändern, deshalb nennen wir nur die wichtigsten:

  1. Sie benötigen Adobe Photoshop Programm. Es wird benutzt, um Quelldateien .PSD zu bearbeiten und Grafik-Design oder Vorlagenbilder zu ändern.
  2. Um die Quelldatei der Vorlage zu bearbeiten, müssen Sie den Code-Editor wie Adobe Dreamweaver, Notepad++, Sublime Text usw. verwenden.
  3. Um die Datei auf den Hosting-Server hochzuladen, benötigen Sie FTP Manager wie Total Commander, FileZilla, CuteFTP usw.
  4. Nutzen Sie die folgenden Tutorials, um Archive mittels WinZip (PC) und StuffitExpander (MAC) zu entpacken.

Hosting

OpenCart basiert sich auf PHP/MySQL Anwendungen, deswegen müssen Sie den Hosting-Server vorbereiten.

Wenn Sie aktiven Server schon haben, müssen Sie ihn prüfen, ob er den Systemanforderungen entspricht und OpenCart Webseiten bedienen kann.

Sie können auch OpenCart auf den PC mittels des lokalen Servers installieren. Um einen lokalen Hosting-Server zu erstellen, müssen Sie die Software wie WAMP, AppServ, MAMP usw. nutzen. Jegliche von ihnen kann wie ein Programm, das OpenCart unterstützt, installiert werden.

Sie können folgende Tutorials nutzen, um den lokalen Server einzustellen:

  • Wie installiert man Web-Entwicklungsumgebung AppServ
  • Wie installiert man Web-Entwicklungsumgebung WAMP
  • Wie installiert man Web-Entwicklungsumgebung XAMP.

Vollständige Installation

Diese Installationsart wird benutzt, wenn Sie keine fertige OpenCart Webseite haben. Das Archiv 'themeXXX(full).zip' enthält Dateien der OpenCart Engine und des Themas mit Demodaten, deswegen müssen Sie OpenCart Engine separat nicht hochladen. Die Demodaten ermöglichen Ihrer Webseite genauso wie die Vorlage auf der Vorschauseite auszusehen (Beispiele der Produkte, Module und Webseiteneinstellungen).

 

Installation auf den existierenden Online-Shop

Folgen Sie dieser Installationsart, wenn Sie eine aktive OpenCart Webseite haben und nur ihr Aussehen ändern möchten, indem Sie ein neues Thema installieren.

 

Vollständige Installation

Vollständige Installation wird dann benutzt, wenn Sie OpenCart Webseite bei Null installieren müssen. Es enthält Durchführung aller vorbereitenden Schritte und Verfügbarkeit der notwendigen Programme.

Erstellen der Datenbank

Nachdem Sie alle Dateien auf den Hosting-Server hochgeladen haben, müssen Sie eine neue Datenbank für OpenCart Webseite vorbereiten. Sie können sie mittels des Datenbank-Verwaltungswerkzeuges im Control Panel des Hostings erstellen. ( normalerweise PhpMyAdmin).

Anhand phpMyAdmin kann man eine neue Datenbank in 3 einfachen Schritten erstellen: (Schauen Sie die folgenden Dias an):

  1. Klicken Sie den Button Databases (Datenbanken) in dem oberen Navigationsmenü, um zur Liste der Datenbanken zu gelangen.
  2. Geben Sie den Namen der Datenbank ins Feld "Create new database (Neue Datenbank erstellen)" ein
  3. Drücken Sie rechts auf den Button Create (Erstellen).

Die Datenbank gibt es jetzt in der Liste. Klicken Sie auf ihren Namen, Zugriff auf sie zu erhalten:

 

  • 1. Öffnen Sie den Tab "Databases (Datenbanken)", um zur Liste der Datenbanken zu gelangen.

  • 2. Geben Sie den Namen der datenbank ein.
    3. Klicken Sie den Button "Erstellen".

  • 4. Die Datenbank gibt es jetzt in der Liste. Klicken Sie auf ihren Namen, Zugriff auf sie zu erhalten.

 

Sie können auch das ausführliche Video-Tutorial nutzenм Wie erstellt man eine Datenbank.

Wenn Sie auf Schwierigkeiten beim Erstellen der Datenbank stoßen, müssen Sie sich in Verbindung mit Ihrem Hosting-Anbieter setzen, damit er Ihnen helfen kann.

Hochladen und Entpacken

Bevor Sie mit OpenCart anfangen zu arbeiten, müssen Sie die Dateien auf den Hosting-Server hochladen. Dies können Sie anhand des Dateimanagers des Hosting oder anhand FTP Manager machen.

  1. Öffnen Sie den Vorlagenordner.
  2. Markieren Sie Dateien 'unzip.php' und 'themeXXX(full).zip' und laden Sie sie auf den Server hoch ( Wie lädt man Dateien auf den Server hoch, Wie lädt man Dateien per cPanel (WebHost Manager, WHM)) hoch.
  3. Geben Sie den Pfad zur Datei 'unzip.php' auf dem Server (http://your_domain_name/unzip.php) im Browser ein.
  4. Jetzt müssen Sie das folgende Fenster sehen:

Bild 1. Startbildschirm des Unzip.php Skripts.

  1. Im Feld 'Choose your zip file (Wählen Sie zip Datei)' wählen Sie hochgeladene 'themeXXX(full).zip' Datei.
  2. Im Feld 'Unzip to (Entpacken in)' legen Sie das Inhaltsverzeichnis fest, wo die Dateien entpackt werden müssen.
  3. Drücken Sie den Button 'Unzip (Entpacken)', um die Dateien zu entpacken.
Legen Sie richtige Erlaubnisse für den Ordner fest, wo die Dateien entpackt werden müssen. Die Erlaubnisse müssen CHMOD 755 oder 777 abhängig von der Serverkonfiguration sein.

OpenCart Installation

Öffnen Sie den Browser und geben Sie den Pfad zum Inhaltsverzeichnis OpenCart in der Adressleiste des Browsers (zum Beispiel http://your_domain_name/opencart) ein. Der Installationsbildschirm OpenCart muss angezeigt werden:

 

1 Installationsschritt Lizenz

 

Bild 1. Installation OpenCart. Schritt 1. Lizenz

Machen Sie sich mit der OpenCart Lizenz bekannt, und wenn Sie den Anforderungen zustimmen, wählen Sie die Option I agree to the license (Ich stimme den Lizenzanforderungen zu) und drücken Sie den Button Continue (Weiter).

 

Schritt2 Vorbereitend

 

Bild 3. OpenCart Installation. Schritt 2. Vorbereitend

Jetzt prüft OpenCart den Hosting-Server: ob seine Einstellungen den OpenCart Anforderungen entsprechen und ob alle Erlaubnisse der Dateien und Inhaltsverzeichnisses richtig festgelegt sind.

Überzeugen Sie sich, dass die Erlaubnisse auf folgende Dateien und Ordner der OpenCart Engine richtig festgelegt sind:

  • chmod 0755 oder 0777 image/
  • chmod 0755 oder 0777 image/cache/
  • chmod 0755 oder 0777 image/data/
  • chmod 0755 oder 0777 system/cache/
  • chmod 0755 oder 0777 system/logs/
  • chmod 0755 oder 0777 download/
  • chmod 0755 oder 0777 config.php
  • chmod 0755 oder 0777 admin/config.php.

 

Schritt 3 Konfiguration

 

Bild 4. OpenCart Installation. Schritt 3. Konfiguration

Jetzt müssen Sie Verbindungsdetails zur Datenbank angeben. In dieser Vorbereitungsphase wurde MySQL Datenbank erstellt und jetzt müssen Sie empfangene/festgelegte von Ihnen Details angeben:

  • Datenbankserver;
  • Datenbankname;
  • Benutzername der Datenbank;
  • Benutzerpasswort der Datenbank;

Wenn Sie auf Schwierigkeiten mit diesen Details stoßen, nehmen Sie den Kontakt mit Ihrem Hosting-Anbieter auf.

Jetzt müssen Sie das Profil des Shopmanagers erstellen. Geben Sie den Namen, das Passwort und die E-Mail-Adresse des Managers ein. Diese Details werden für den Zugang zum Control Panel OpenCart benutzt.

 

Schritt 4 Fertig!

 

Bild 5. OpenCart Installation. Schritt 4. Fertig!

Das ist der letzte Installationsschritt der OpenCart Engine. Es gibt zwei Optionen: man kann entweder zur Webseite des OpenCart Online-Shops oder zum Control Panel OpenCart gelangen.

Sie können auch das Tutorial zur Installation nutzen: OpenCart. Wie installiert man OpenCart Engine und Vorlage (anhand “fullpackage”) .

Löschen Sie den Ordner "install" auf dem Server. Sie können sich in den Online-Shop nicht einloggen, bis dieser Ordner auf dem Server ist.

Installieren Sie Erlaubnisse 644 oder 444 auf existierende Dateien, um den Zugriff auf diese Dateien zu beschränken:

  • config.php
  • index.php
  • admin/config.php
  • admin/index.php
  • system/startup.php.

Installation wird abgeschlossen. Um den Online-Shop zu öffnen, geben Sie Ihre Domainnamen in der Adressleiste des Browsers ein. Um den Zugriff zum Control Panel OpenCart zu erhalten, geben Sie Domain_Name/admin ein.

Installation auf den existierenden Online-Shop

Überzeugen Sie sich, dass die Version Ihrer OpenCart Engine den Anforderungen der Vorlage entspricht. Alle OpenCart Versionen sind auf der offiziellen OpenCart Webseite verfügbar.

  1. Um OpenCart Vorlage zu installieren, müssen Sie ihre Dateien auf en Hosting-Server hochladen. Im Vorlagenordner OpenCart finden Sie das Archiv "themeXXX(update).zip" und laden Sie es ins Stamminhaltsverzeichnis der installierten OpenCart Vorlage hoch.
    Sie können auch das ausführliche Video-Tutorial nutzen, wie lädt man Dateien auf den Hosting-Server hoch.
    Vorlagendateien werden in keinster Weise den Online-Shop beschädigen.
  2. Dann laden Sie die Datei unzip.php der Vorlage auf den Server in in demselben Inhaltsverzeichnis wie vorheriges Archiv hoch.
  3. Dann öffnen Sie die Webseite im Browser und fügen Sie - unzip.php zur URL hinzu. Die Adresse wird so aussehen: yourwebsite.com/unzip.php.
  4. In der Liste "Choose your zip file" (Wählen Sie Ihre zip Datei), markieren Sie die Datei themeXXX(update).zip und drücken Sie "Unzip" (Entpacken). Dann wird das Entpacken der Dateien auf den Server in demselben Inhaltsverzeichnis begonnen. Dieser Prozess braucht Zeit.
  5. Geben Sie yourwebsite.com/install/ in der Adressleiste des Browsers ein .
  6. Zum Schluss folgen Sie den Anleitungen zur Installation, die Sie auf dem Bildschirm sehen.
  7. Loggen Sie ins Control Panel ein, gehen Sie ins Menü Extensions -> Extension Installer und laden Sie die Dateien aus dem Archiv themeXXX(extensions) hoch. Nachdem Sie alle Dateien hochgeladen haben, gehen Sie ins Menü Extensions -> Modifications und drücken Sie auf den Button refresh in der oberen rechten Ecke des Bildschirmes.

Sie können auch das ausführliche Video-Tutorial zur Installation nutzen: OpenCart. Wie installiert man die Vorlage auf den existierenden Online-Shop (anhand des Archivs “update”).

Validation

In unseren Vorlagen befolgen wir semantischen Regeln für das Schreiben vom Code HTML-Markup und den CSS-Stilen. Aber einige Fehler können sowieso bei W3C-Validierung auftreten. Erstellen des zu 100 Prozent korrekten Codes entsprechend den Standards vom W3C Consortium beschränkt die Verwendung von modernen Technologien zum Erstellen von Websites wie Spezifikation CSS3 und HTML5 Markup. Das Ziel unseres Unternehmens liegt darin, dass wir reiche Benutzererfahrung mit hochwertigen Vorlage zur Verfügung stellen möchten. Aus diesem Grund befolgen wir nicht allen Regeln.
