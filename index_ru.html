Введение Что находится в шаблоне и для чего его можно использовать

Благодарим Вас за приобретение шаблона OpenCart. Данная документация состоит из нескольких частей и охватывает весь процесс установки и настройки веб-сайта OpenCart c нуля. Мы сделали все возможное, чтобы это руководство было понятным и удобным в использовании.

Что представляет собой OpenCart?

OpenCart - это бесплатная платформа электронной коммерции с открытым исходным кодом. OpenCart позволяет создать профессиональную и надежную основу для построения успешного онлайн магазина. Сообщество Opencart включает множество пользователей, начиная с опытных веб-разработчиков, ищущих удобный интерфейс, и заканчивая владельцами магазинов, впервые запускающими онлайн-бизнес. OpenCart предоставляет огромное количество функций, позволяющих полностью контролировать магазин. С инструментами OpenCart Вы легко раскроете потенциал магазина. Подробнее

Структура файлов

Шаблон состоит из следующих папок и файлов:

  • documentation - содержит детальное руководство по установке движка и шаблона;
    • documentation.html - основной файл документации. Вы его сейчас читаете :)
  • screenshots - содержит скриншоты темы;
  • themeXXX - включает в себя:
    • themeXXX(full).zip - архив с движком OpenCart, шаблоном и демо-данными темы (предназначен для установки с нуля);
    • themeXXX(update).zip - архив с шаблоном без демо-данных (для установки поверх существующего магазина);
    • themeXXX(extensions).zip - архив с xml-файлами;
  • info.txt - инструкции по разархивированию исходных файлов;
  • sources_XXXXXXXX.zip - архив с исходными файлами. Включает:
    • psd - содержит исходный файл шаблона в формате Adobe Photoshop (.psd);
    • SCSS - содержит исходные файлы CSS-стилей темы написанных с помощью SCSS.

Подготовка Что нужно сделать перед началом работы

Прежде, чем перейти к установке веб-сайта OpenCart, нужно полностью подготовиться.

Программное обеспечение

Для полноценной работы с шаблоном OpenCart мы рекомендуем скачать все требуемые приложения. Список необходимого программного обеспечения находится на странице предпросмотра шаблона. Требования для различных шаблонов могут меняться, поэтому мы перечислим лишь основные:

  1. Вам может понадобиться программа Adobe Photoshop. Она используется для редактирования исходных файлов .PSD и необходима, если требуется изменить графический дизайн и изображения шаблона.
  2. Для редактирования исходного кода файлов шаблона нужно использовать редактор кода, например Adobe Dreamweaver, Notepad++, Sublime Text и т.п.
  3. Чтобы загрузить файлы на хостинг сервер, потребуется FTP менеджер, например Total Commander, FileZilla, CuteFTP и т.д.
  4. Воспользуйтесь следующими инструкциями, чтобы распаковать архивы при помощи WinZip(PC) и StuffitExpander(MAC).

Хостинг

Поскольку OpenCart базируется на приложениях PHP/MySQL, необходимо подготовить хостинг сервер.

При наличии активного сервера, нужно проверить его на соответствие системным требованиям и на способность обслуживать веб-сайты OpenCart.

Также можно установить OpenCart на персональном компьютере при помощи локального сервера. Для создания локального хостинг сервера необходимо использовать программное обеспечение WAMP, AppServ, MAMP и т.п. Любое из них устанавливается как обычная программа и поддерживает OpenCart.

Вы можете воспользоваться следующими инструкциями для настройки локального сервера:

  • Как установить среду веб-разработки AppServ
  • Как установить среду веб-разработки WAMP
  • Как установить среду веб-разработки XAMP.

Начало использования Куда нажимать, чтобы приступить к работе

Полная установка

Данный тип установки используется, если у вас нет готового сайта OpenCart. Архив 'themeXXX(full).zip' включает файлы движка OpenCart и темы с демо-данными, поэтому вам не нужно отдельно загружать движок OpenCart. Демо-данные позволят Вашему сайту выглядеть так же, как и шаблон на странице предпросмотра, со всеми образцами товаров, модулей и настройками сайта.

 

Установка на существующий магазин

Следуйте данному типу установки, если у вас уже есть действующий сайт OpenCart и требуется всего лишь изменить его внешний вид, путем установки новой темы.

 

Полная установка

Полная установка используется в том случае, если нужно установить веб-сайт OpenCart c нуля. Предусматривается предварительное выполнение всех подготовительных шагов и наличие необходимых программ.

Создание базы данных

Когда все файлы будут загружены на хостинг сервер, необходимо подготовить новую базу данных для веб-сайта OpenCart. Ее можно создать через инструмент управления базами данных в контрольной панели хостинга (обычно PhpMyAdmin).

При помощи phpMyAdmin можно создать новую базу данных за 3 простых шага: (см. следующие слайды):

  1. Щелкните кнопку Databases (базы данных) в верхнем меню навигации, чтобы перейти в список баз данных.
  2. Введите название базы данных в поле "Create new database (Создать новую базу данных)".
  3. Нажмите справа на кнопку Create (Создать).

База данных появилась в списке. Кликните по ее названию, чтобы получить доступ к ней:

 

  • 1. Откройте вкладку "Databases (Базы данных)", чтобы перейти к списку баз данных.

  • 2. Введите название базы данных.
    3. Щелкните кнопку "Создать".

  • 4. База данных появилась в списке. Кликните по названию, чтобы получить доступ к базе данных.

 

Вы также можете воспользоваться детальной видео инструкцией Как создать базу данных.

В случае возникновения сложностей при создании базы данных, необходимо обратиться к хостинг провайдеру за поддержкой.

Загрузка шаблона

Для начала работы с OpenCart, требуется загрузить файлы на хостинг сервер. Это можно сделать при помощи файлового менеджера хостинга или альтернативного FTP менеджера.

  1. Откройте папку шаблона.
  2. Перейдите в папку 'themeXXX' и распакуйте архив 'themeXXX(full).zip' на Ваш компьютер. Содержимое архива загрузите на сервер ( Как загрузить файлы на сервер, Как загрузить файлы с помощью cPanel (WebHost менеджер, WHM)).
  3. Задайте правильные разрешения на следующие папки и файлы движка OpenCart:
    • system/cache/;
    • system/logs/;
    • system/download/;
    • system/upload/;
    • image/;
    • image/cache/;
    • image/catalog/;
    • config.php;
    • admin/config.php.

    Разрешения должны быть CHMOD 755 или 777 в зависимости от конфигурации сервера. Для этого выделите нужные файлы и папки и если Вы работаете в файловом менеджере - найдите кнопку "изменить права". Если Вы работаете под Linux - вам поможет команда chmod -R 777 "имя_папки".

Установка OpenCart

Откройте браузер и пропишите путь к директории OpenCart в адресной строке браузера (например, http://your_domain_name/opencart). Должен появиться экран установки OpenCart:

 

1 этап установки Лицензия

 

Рис 1. Установка OpenCart. Этап 1. Лицензия

Ознакомьтесь с лицензией OpenCart и, если Вы согласны с ее условиями, нажмите кнопку Continue (Продолжить).

 

2 этап Предварительный

 

Рис 2. Установка OpenCart. Этап 2. Предварительный

На данном этапе, OpenCart проверит хостинг сервер: соответствуют ли настройки требованиям OpenCart и правильно ли заданы разрешения файлов и директорий.

 

3 этап Конфигурация

 

Рис 3. Установка OpenCart. Этап 3. Конфигурация

На данном этапе нужно указать детали подключения к базе данных. Насколько Вы помните, на этапе подготовки была создана база данных MySQL, сейчас нужно указать полученные/заданные вами детали:

  • сервер базы данных;
  • название базы данных;
  • имя пользователя базы данных;
  • пароль пользователя базы данных.

При возникновении сложностей с этими деталями, обратитесь к хостинг провайдеру.

Также нужно создать профиль администратора магазина. Введите имя, пароль и e-mail администратора. Эти детали будут использоваться для доступа в панель управления OpenCart.

 

4 этап Готово!

 

Рис 4. Установка OpenCart. Этап 4. Готово!

Это заключительный этап установки движка OpenCart. Предлагаются две опции: перейти на Ваш сайт магазина OpenCart или в панель управления OpenCart.

Вы также можете воспользоваться инструкцией по установке: OpenCart. Как установить движок и шаблон OpenCart (при помощи “fullpackage”).

Удалите папку "install" с сервера. Вы не сможете войти в магазин, пока эта папка находится на сервере.

Установите разрешения 644 или 444 на следующие файлы с целью ограничения доступа к ним:

  • config.php;
  • index.php;
  • admin/config.php;
  • admin/index.php;
  • system/startup.php.

Установка завершена. Чтобы открыть магазин, пропишите свое доменное имя в адресной строке браузера. Для доступа в панель управления OpenCart, пропишите доменное_имя/admin.

Установка на существующий магазин

Убедитесь, что Ваша версия движка OpenCart соответствует требованиям шаблона. Все версии OpenCart доступны на официальном сайте OpenCart .

  1. Чтобы установить шаблон OpenCart, нужно загрузить его файлы на хостинг сервер. В папке OpenCart шаблона найдите архив "themeXXX(update).zip" разархивируйте его на свой компьютер.
  2. После этого загрузите разархивированные файлы в корневую директорию установленного OpenCart.
    Вы также можете воспользоваться детальной видео инструкцией, как загружать файлы на хостинг сервер.
    Файлы шаблона никаким образом не повредят магазину;
  3. Затем откройте сайт в браузере и добавьте /install в конце адресной строки браузера (например, http://your_domain_name/opencart/install)
  4. В появившейся на экране странице нажмите кнопку Continue (Продолжить). Обновление темы может занять некоторое время.
  5. Войдите в панель управления Вашим сайтом, перейдите в меню Расширения -> Установщик расширений (Extensions -> Extension Installer) и загрузите файлы из архива themeXXX(extensions). После загрузки всех файлов архива, перейдите в меню Расширения -> Модификации (Extensions -> Modifications) и нажмите копку обновить(refresh) в правом верхнем углу экрана;
  6. Далее перейдите в настройки магазина Система -> Настройки (System -> Settings), и во вкладке Магазин (Store) в поле Шаблон (Template) выберите тему - themeXXX.

Вы также можете воспользоваться инструкцией по установке: OpenCart. Как установить шаблон на существующий магазин (при помощи архива “update”).

Настройка расширений

Установка модулей

Установка модуля проходит в несколько шагов. Перейдите в панель администратора (добавив /admin в адресной строке - www.yoursite.com/admin). Далее, в боковой панели выберите вкладку Расширения (Extensions) и в выпадающем списке Модули (Modules) (Рис.5). В открывшемся окне показаны все установленные и готовые к установке модули. Если напротив названия модуля есть кнопка Деинсталлировать (Uninstall) - это значит, что модуль уже установлен и, нажав на Редактировать (Edit), Вы можете редактировать настройки данного модуля. В противном случае, нажав на Установить (Install), Вы произведете установку модуля. Удаление модуля происходит похожим образом: необходимо нажать кнопку Деинсталлировать (Uninstall), после чего модуль будет удален.

Будьте внимательны, так как при удалении модуля будут удалены его настройки, которые восстановить будет невозможно.

Рис 5. Установка модуля

После установки модуля необходимо его настроить. Настройки могут отличаться в зависимости от модуля, кроме обязательного поля Status (Рис. 6) - данное поле включает/выключает наш модуль.

Рис 6. Редактирование модуля

Для добавления модуля на страницу требуется перейти к списку страниц. Это можно сделать 2-мя способами (Рис.7):

  1. В боковом меню перейдите в раздел Система -> Дизайн -> Макет (System -> Design -> Layout);
  2. Находясь в разделе модулей, нажмите на ссылку here.

Рис 7. Переход на выбор страницы для редактирования

Выберите из списка страницу, на которую нужно добавить модуль, напротив неё нажмите Редактировать (Edit) (Рис.8).

Рис 8. Выбор страницы для редактирования

Далее показан список модулей, которые уже имеются на странице, и их позиции (Рис.9).

Рис 9. Добавление модулей на страницу

Доступны следующие позиции (по порядку отображения):

  1. Header Top (после секции Header);
  2. Column-left (левая боковая панель);
  3. Column-right (правая боковая панель);
  4. Content-top;
  5. Content-bottom;
  6. Footer.

Поле Порядок Сортировки (Sort Order) указывает порядок отображения модулей на странице, которые находятся в одной позиции. Для добавления модуля на страницу нажмите Добавить модуль (Add Module). В появившейся строке выберите необходимый модуль и позицию на странице и нажмите Сохранить (Save). Установка модуля на этом закончена.

Создание баннеров

Модификация TM Description for banners

Модификация предназначенная для добавления описания баннеров. Для удаления или отключения модификации зайдите в Расширения -> Модификации (Extensions -> Modifications) найдите в списке "TM Description for banners" расширение и нажмите кнопку Отключить (Disable) или Удалить (Delete).

Настройки некоторых модулей содержат поле для выбора баннера, который будет использоваться. Перейдите в раздел Система -> Дизайн -> Баннера (System -> Design -> Banner) (Рис.10).

Рис 10. Страница баннеров

На данной странице отображается список созданных баннеров. Для создания нового баннера нажмите Добавить новый (Add New), для редактирования существующего баннера нажмите Редактировать (Edit).

Рис 11. Редактирование баннера

Далее укажите Banner Name, Status и нажмите Добавить баннер (Add Banner). В появившейся таблице укажите:

  1. Название баннера;
  2. Описание - отображаемый в баннере текст (поддерживается html-разметка);
  3. Ссылку - укажите относительную ссылку. ссылку относительно адреса сайта (например, index.php?route=product/product&product_id=47) данный параметр не является обязательным;
  4. Изображение;
  5. Порядок баннера - так как баннер может быть не один, необходимо заполнять данное поле.

Для удаления баннера нажмите Удалить (Remove). После создания баннера, Вы сможете использовать его для других модулей.

Parallax

Параллакс-скроллинг - это специальная техника скроллинга в компьютерной графике, при которой изображения на заднем фоне двигаются медленнее контента, который находится на переднем фоне, создавая, таким образом, иллюзию глубины и эффект погружения в двухмерной видео-игре. Данный модуль добавит параллакс-эффект на сайт, не требуя каких-либо дополнительных навыков и усилий.

После установки модуля Вы можете настроить его на странице настроек модуля (Рис.12).

Рис 12. Настройка модуля Parallax

Настройки:

  1. Module Name - название модуля;
  2. Banner - выберите из списка созданных ранее баннеров;
  3. Width - ширина картинки для модуля;
  4. Height - высота картинки для модуля;
  5. Status - статус модуля (включен/отключен).
Ширина и высота картинки должна соответствовать оригинальной (используемой по умолчанию), иначе эффект может не соответствовать Вашим ожиданиям.

TM Category

Данный модуль позволяет выводить категории в виде аккордеон меню, также выводит все подкатегории (Рис.13). Как правило, используется в боковой панели.

Рис 13. Внешний вид модуля TM Category

Настройки:

После установки модуля Вы можете включить или выключить его на странице конфигураций.

TM Facebook Box

Данный модуль отображает Facebook like box - отображает лица и количество поклонников, кому естественно нравится Ваш сайт или бренд (Рис.14).

Рис 14. Внешний вид модуля TM Facebook

После установки модуля Вы можете настроить его на странице настроек модуля (Рис.15).

Рис 15. Настройка модуля TM Facebook

Настройки:

  1. Module Name - название модуля;
  2. Facebook Page URL - ссылка на Facebook страницу;
  3. Color Scheme - цветовая схема (светлая, темная);
  4. Show Faces - показывать лица пользователей, которым понравилась страница (yes, no);
  5. Show Stream - показывать ленту последних постов (yes, no);
  6. Show Header - показывать заголовок блока (yes, no);
  7. Show Border - показывать границу блока (yes, no);
  8. Width - ширина блока в пикселях;
  9. Height - высота блока в пикселях;

TM Cookie Policy

Данный модуль оповещает пользователя об использовании Кукис (cookies) (Рис. 27). Согласно Закону о Кукис, модуль позволяет оповещать пользователей об использовании Кукис.

Cookies - это небольшие текстовые файлы, которые веб-сайты создают на компьютерах пользователей. Они используются для идентификации пользователей и улучшения качества использования веб-сайта. Cookies являются важной частью многих функций веб-сайтов.

Например, cookies позволяют онлайн магазинам запоминать, какие товары Вы добавили в корзину. Также они сохраняют данные входа на сайт или предоставляют ценную статистику использования и информацию владельцам сайтов.

Если у вас есть веб-сайт, он почти наверняка использует cookies. Их главная функция - изучать количество и интересы пользователей через специальные инструменты, например Google Analytics. Также они могут использоваться для отображения объявлений, или – если у вас есть онлайн-магазин – быть ключевой частью системы электронной коммерции.

26 мая 2012 г. - ICO опубликовал акт о том, что "подразумеваемое согласие" не допускается и необходимо сообщать о политике приватности/cookies.
